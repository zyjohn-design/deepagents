## 区域与线路提取约束

### 管控区 boundaries 约束

管控区是一个**合围区域**（封闭多边形），boundaries 必须满足以下规则：

#### 1. 合围闭合性
- boundaries 描述的道路必须构成一个**首尾相连的封闭环**
- 原文中通常以"合围区"、"合围区域"、"合围区内道路"结尾
- 提取时必须保留完整的合围描述，不可截断

#### 2. 边界道路完整性
- 必须提取构成合围边界的**所有道路名称**，不可遗漏
- 每条边界道路的范围标注必须保留，如"沿江大道(民生路至一元路)"
- 含/不含标注必须保留，如"一元路(不含)"、"黄石路(含)"
- "含"表示该道路本身在管控范围内，"不含"表示该道路是边界但本身不在管控范围内

#### 3. 嵌套层级关系
- 三大管控区呈**同心圆嵌套**关系：核心管控区 ⊂ 中端疏导区 ⊂ 远端分流区
- 核心管控区的外边界 = 中端疏导区的内边界
- 如果文档只描述了外边界，内边界从嵌套关系推断

#### 4. 提取格式
```
正确: "沿江大道(民生路至一元路)、一元路(不含)、胜利街(一元路至黄石路,不含)、黄石路(不含)、中山大道(黄石路至民生路)、民生路(不含)合围区内道路"
错误: "沿江大道附近区域"          ← 过于模糊，不是合围描述
错误: "沿江大道、一元路"           ← 不完整，未构成闭合环
错误: "核心管控区"                 ← 这是 area_name，不是 boundaries
```

### 跨年夜场景无线路类型

跨年夜活动不涉及比赛路线，所有管控对象都是**合围区域**，不应出现"线路"类型。
文档中提到的具体路段（如"沿江大道"、"中山大道"）是合围区域的**边界道路**，不应单独作为区域提取，而应归纳到所属的管控区 boundaries 中。

## 阶段详细说明

### 1. 启动准备阶段 (preparation)

**典型任务**:
| 时间点 | 任务类型 | 示例 |
|-------|---------|------|
| 15:00 | 指挥部启动 | 各级指挥部指挥长进驻运行 |
| 15:00 | 车辆管控 | 核心管控区禁止机动车停放 |
| 16:00 | 场地准备 | 完成路面和场地清理 |
| 16:30 | 力量部署 | 安保力量全部到岗 |
| 17:00 | 保障就位 | 协同保障单位力量投放到位 |

### 2. 管控实施阶段 (implementation)

**典型任务**:
| 时间点 | 任务类型 | 示例 |
|-------|---------|------|
| 17:30 | 分流启动 | 启动疏导区、分流区分流措施 |
| 18:30 | 核心管控 | 核心区禁止机动车通行 |
| 22:00 | 疏散准备 | 疏散车辆、共享单车投放 |
| 23:30 | 车辆集结 | 公交/出租车完成停候排队 |
| 23:45 | 临时管控 | 重点路口启动临时交通管控 |
| 00:00 | 人群疏散 | 全区域同步实施人群疏散 |

### 3. 疏散收尾阶段 (dispersal)

**典型任务**:
| 时间点 | 任务类型 | 示例 |
|-------|---------|------|
| 02:00 | 逐步恢复 | 视情逐段恢复沿江大道车辆放行 |
| 03:00 | 全面恢复 | 中山大道、京汉大道恢复放行 |
| 03:00+ | 常态交通 | 恢复常态交通和信号灯配时 |

### 4. 全时段保障 (continuous)

**典型任务**:
| 类型 | 示例 |
|-----|------|
| 应急通道 | 沿江大道应急通道全时段硬隔离 |
| 医疗急救 | 8台医疗急救车辆现场待命 |
| 消防备勤 | 22个前置执勤点消防备勤 |

## 专业术语表

### 管控措施术语
| 术语 | 含义 |
|-----|------|
| 只出不进 | 区域内车辆可驶出，禁止新车驶入 |
| 净空管理 | 清空并保持无车状态 |
| 单向循环 | 按指定方向单向通行 |
| 梯次分流 | 按距离远近分级引导分流 |
| 滴水式放行 | 缓慢逐辆放行 |
| 上满即走 | 车辆载满乘客立即发车 |

### 车辆类型
| 类型 | 说明 |
|-----|------|
| 巡游出租车 | 有顶灯及车身喷涂标识的出租车 |
| 网约车 | 线上预约车辆 |
| 智能网联车 | 萝卜快跑等自动驾驶车辆 |
| 疏散摆渡车 | 公交疏散专用车辆 |
